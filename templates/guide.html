Guide BaseLIR

BaseLIR est une application de gestion des habilitations individus-rôles. Elle remplace un fichier Excel et offre un suivi des attestations, un historique des modifications et une gestion des accès par rôle.

Installation

Prérequis

  • Python 3.10 ou supérieur
  • pip
  • Accès Internet au premier lancement (CDN Tailwind CSS)

1. Installer les dépendances

python -m venv venv
venv\Scripts\activate   # Windows
pip install -r requirements.txt

2. Lancer l'application

python run.py

La base de données SQLite est créée automatiquement dans data/baselir.db. Deux comptes par défaut sont initialisés au premier démarrage.

3. Accéder à l'application

Ouvrez votre navigateur sur http://localhost:8001

Habilitations

La liste des habilitations affiche l'ensemble des personnes habilitées. Chaque ligne indique le statut (coloré), le rôle, le domaine et les dates clés. Les dates d'attestations dépassées apparaissent en rouge.

Champs d'une habilitation

Nom et PrénomIdentité de la personne habilitée
StatutÉtat de l'habilitation (ex : Actif, Suspendu, Révoqué) — couleur personnalisable
Filiale du GroupeFiliale de rattachement
DescriptionNature de la mission ou du profil
ServiceService ou département d'appartenance
SociétéEntité juridique de rattachement
RôleRôle ou niveau d'accès accordé
DomainePérimètre fonctionnel de l'habilitation
Date d'octroiDate à laquelle l'habilitation a été accordée
Date des attestationsDate limite de validité — génère un écart si dépassée
Champs perso.Champs supplémentaires définis par l'administrateur (ex : Région, Niveau…)

Créer une habilitation

  1. Cliquer sur + Nouvelle habilitation
  2. Remplir les champs (les listes déroulantes sont issues des référentiels)
  3. Cliquer sur Enregistrer

Modifier ou supprimer

Depuis la fiche d'une habilitation, le responsable peut modifier tous les champs ou supprimer l'entrée. Chaque modification est tracée dans l'historique.

Recherche et filtres

La liste dispose d'une barre de recherche (nom/prénom) et de filtres par statut, domaine, rôle et société. Les colonnes sont triables.

Historique des modifications

Chaque création, modification et suppression est enregistrée automatiquement. L'historique est accessible depuis la fiche de l'habilitation et indique :

  • L'action effectuée (Création, Modification, Suppression)
  • L'utilisateur responsable de la modification
  • La date et l'heure exactes
  • Les valeurs avant et après modification

Audit — Écarts d'attestations

La page Audit liste toutes les habilitations dont la date des attestations est antérieure à la date du jour. Ces écarts signalent des habilitations à régulariser.

Comment régulariser un écart

  1. Cliquer sur Régulariser à droite de la ligne
  2. Modifier la date des attestations (nouvelle échéance) ou changer le statut (Suspendu, Révoqué)
  3. Enregistrer — l'écart disparaît automatiquement si la nouvelle date est dans le futur

Filtres disponibles

La liste d'écarts peut être filtrée par domaine, service et société pour faciliter le traitement par périmètre.

Les auditeurs peuvent consulter et filtrer la liste des écarts. Seuls les responsables peuvent effectuer les corrections (lien "Régulariser").

Référentiels

Les référentiels définissent les valeurs disponibles dans les listes déroulantes des formulaires. Chaque référentiel est géré dans un onglet dédié.

RéférentielParticularité
StatutsCouleur associée à chaque statut pour la visualisation dans la liste
FilialesFiliales du groupe
DescriptionsNatures de mission ou profils
ServicesServices et départements
SociétésEntités juridiques
RôlesRôles ou niveaux d'accès
DomainesPérimètres fonctionnels

Pour chaque référentiel : ajouter, modifier, supprimer et définir l'ordre d'affichage. Les valeurs utilisées dans des habilitations existantes ne peuvent pas être supprimées.

Champs personnalisés

L'onglet Champs perso. dans Référentiels permet de créer des champs supplémentaires (ex : Région, Pays, Niveau de certification…) sans modifier le schéma de la base de données. Ces champs apparaissent automatiquement dans tous les formulaires d'habilitation.

Créer un champ personnalisé

  1. Aller dans Référentiels → Champs perso.
  2. Saisir un libellé (ex : Région) et un nom technique (ex : region)
  3. Créer le champ — il apparaît immédiatement dans les formulaires d'habilitation
  4. Ajouter les valeurs possibles (ex : Nord, Sud, Est, Ouest)
Supprimer un type de champ personnalisé supprime également toutes ses valeurs et les données associées aux habilitations existantes.

Import CSV / Excel

Importer en masse depuis un fichier CSV ou Excel existant (extraction RH, tableur legacy…). Les valeurs de référentiels absentes sont créées automatiquement.

Procédure

  1. Aller dans Import
  2. Télécharger le modèle CSV si nécessaire (lien en bas de page)
  3. Ou utiliser le fichier sample_import.xlsx fourni
  4. Sélectionner le fichier (.csv, .xlsx)
  5. Cliquer sur Importer — un rapport s'affiche avec le détail des lignes traitées

Colonnes attendues

ColonneNoms acceptés
Nom et PrénomNom et Prénom, Nom Prénom, nom_prenom
StatutStatut
Filiale du GroupeFiliale du Groupe, Filiale
DescriptionDescription
ServiceService
SociétéSociété, Societe
RôleRôle, Role
DomaineDomaine
Date d'octroiDate d'octroi, Date octroi, date_octroi
Date des attestationsDate des attestations, Date attestation

L'ordre des colonnes est libre. Les noms de colonnes sont normalisés (accents et casse ignorés).

Formats de dates acceptés : JJ/MM/AAAA, AAAA-MM-JJ, JJ-MM-AAAA.

Les lignes en doublon (même nom + même société) sont ignorées sans erreur.

Gestion des utilisateurs

Accessible depuis le menu Utilisateurs (responsables uniquement). Permet de créer, modifier et supprimer les comptes applicatifs.

Créer un compte

  1. Cliquer sur Nouvel utilisateur
  2. Saisir un identifiant, un mot de passe et choisir le rôle
  3. Enregistrer

Modifier un compte

Le mot de passe peut être laissé vide pour le conserver sans modification.

Il est impossible de supprimer son propre compte ou le dernier compte responsable.

Journal d'activité

Toutes les actions sont tracées automatiquement : connexions, déconnexions, créations, modifications, suppressions, imports et changements de configuration. Le journal est accessible à tous les utilisateurs authentifiés et consultable à tout moment.

Paramètres

Accessible depuis le menu Paramètres (responsables uniquement). Trois sections : Thème, Logo et Active Directory.

Thème de couleur

Personnalisez l'interface depuis la section Thème :

  • Couleur principale — barre de navigation et boutons principaux (8 choix)
  • Couleur secondaire — accents de l'interface (8 choix)

Le thème est enregistré en base et s'applique immédiatement à toute l'interface, y compris la page de connexion.

Logo société

Déposez un logo (PNG, JPG, SVG, GIF, WEBP) depuis la section Logo. Il s'affiche sur la page de connexion à la place de l'icône par défaut.

SSO Active Directory / LDAP

  1. Cocher Activer la connexion Active Directory
  2. Renseigner le serveur LDAP/AD (ex : ad.societe.local), le port, le domaine et la base DN
  3. Optionnel : activer TLS (port 636)
  4. Optionnel : OU autorisée — seuls les utilisateurs dont le DN contient ce chemin peuvent se connecter (ex : ou=IT,dc=societe,dc=local)
  5. Optionnel : Groupe AD autorisé — seuls les membres de ce groupe peuvent se connecter
  6. Enregistrer

En cas d'échec AD (serveur indisponible, utilisateur hors OU ou groupe), la connexion locale reste disponible.

Les comptes AD inconnus sont créés automatiquement avec le rôle Auditeur. Le responsable peut modifier leur rôle depuis la page Utilisateurs.

Déploiement Docker

Démarrage

docker compose up --build -d

Application accessible sur http://localhost:8001

Persistance des données

Deux volumes montés depuis le dossier du projet :

  • ./data — base SQLite et configuration
  • ./uploads — logo société

Les données survivent aux redémarrages et mises à jour du conteneur.

Sécurité en production

Remplacer la clé de session dans app/main.py par une clé aléatoire :

python -c "import secrets; print(secrets.token_hex(32))"

Données de démonstration

Charger les données fictives

python seed_demo.py

Injecte 20 habilitations fictives (Groupe Meridia) avec des statuts variés et des écarts d'attestations.

Remettre la base à zéro

python reset_db.py

Supprime toutes les habilitations, référentiels et configuration, puis recrée les comptes et données par défaut.
